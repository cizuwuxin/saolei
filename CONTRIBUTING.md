# Contributing

Thanks for helping improve Minesweeper.

## Guidelines

- Keep changes focused and easy to review.
- Preserve the single-file deployment goal unless a refactor clearly improves maintainability.
- Keep the UI responsive and mobile-friendly.
- Update the README when gameplay or controls change.
- Test both desktop and mobile interactions when possible.

## Good PRs usually include

- A short description of the change
- Before/after screenshots for visual updates
- Notes about performance or compatibility impact
